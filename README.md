# NKU Rust Project

基于 `cc-haha` 思路实现的 Rust 版 Agent CLI 项目。

本项目面向命令行场景，目标是用 Rust 重构一部分适合 CLI 的核心能力，并在“代码版本回滚”方向上做出一个可演示、可说明的创新功能。

## 项目简介

随着智能编程助手逐渐深入日常开发，如何在命令行环境中提供稳定、轻量、可控的 Agent 能力，成为一个很有价值的实践方向。本项目选择将原项目中较适合迁移到 Rust 的功能模块进行重构，重点关注：

- 稳定的命令行入口
- 基础会话与状态管理
- 常用文件操作工具
- 面向代码修改场景的版本回滚

相比直接实现复杂的桌面端能力，本项目更强调 CLI 场景下的工程可维护性、执行效率与可演示性。

## 项目目标

1. 使用 Rust 构建一个可运行的 Agent CLI 框架。
2. 迁移若干低耦合、易验证的核心功能。
3. 设计并实现一个代码版本回滚创新模块。
4. 形成适合课程展示、代码讲解和后续扩展的工程结构。

## 功能规划

### 基础功能

- CLI 启动与参数解析
- 配置文件与环境变量加载
- 会话记录与本地持久化
- 文件读取、写入、编辑、搜索
- 命令执行与结果回传

### 创新功能

- 代码版本回滚
  - 自动快照
  - diff 记录
  - 回滚预览
  - 按文件或按步骤恢复

## 技术选型

- 语言：Rust
- 构建工具：Cargo
- 版本管理：Git
- 核心思路：模块化拆分、trait 抽象、状态持久化、快照式回滚

### 模块说明

- `cli`：程序入口、命令路由、参数解析、配置加载
- `core`：会话、消息、状态、provider 抽象
- `tools`：基础文件与命令工具
- `rollback`：快照、diff、恢复、历史管理
- `tests`：集成测试与回归测试

## 团队分工

### 成员 A：CLI 与工程骨架

负责入口、参数、配置、日志和整体启动流程。  
主要参考原项目的入口链路与初始化逻辑，形成 Rust 版最小可运行框架。

### 成员 B：核心状态与会话

负责 session、历史记录、状态持久化与 provider 抽象。  
目标是保证 Agent 在多轮交互中具备稳定上下文。

### 成员 C：基础工具层

负责文件读写、编辑、搜索、命令执行等工具能力。  
这是 Agent 真正具备“动手修改代码”的关键基础。

### 成员 D：版本回滚创新

负责快照、diff、回滚预览和恢复逻辑。  
这是本项目的重点创新模块，也是最终展示的核心亮点。

### 成员 E：测试与集成

负责端到端测试、回归测试、文档整理和演示准备。  
保证项目结构清晰、运行稳定、展示完整。

## 设计思路

本项目采用“底座优先”的实现方式：

1. 先完成 CLI 骨架和配置加载。
2. 再补齐会话状态和基础工具。
3. 最后实现回滚系统并打通完整链路。

这样可以先保证项目可运行，再逐步增加复杂能力，降低整体实现风险。

## 运行方式

项目完成后可通过以下方式运行：

```bash
cargo build
cargo run -- --help
```

后续将根据实际命令设计补充更多示例。
